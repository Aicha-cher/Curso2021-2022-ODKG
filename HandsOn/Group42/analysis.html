Dataset analysis

There are 2245 records in our dataset in summary. The reports are from 2019 Januar 1st to 2021 June 30th.

Column Type Comments/Range Problems
Nº EXPEDIENTE Text The identification number of the accident. Not unique (there might be different people in the same accident) Duplicated values because of having more people in the same accident
FECHA Text Date of the accident, DD/MM/YYYY format. No missing values.
HORA Text Exact time of the accident, HH:MM format. No missing values. 458 different values.
CALLE Text The street where the accident happened. There are some ","s in between the values which needs to be corrected. This attribut does not have a well defined format. The value is not always a street, it could be a name of a restaurant is the accident happened in front of that restaurant.
NÚMERO Text The number of the street where the accident happend. There are 1075 missing values.
DISTRITO Text The distinct of the accident. 21 different values (no missing values).
TIPO ACCIDENTE Text The type of the accident. 11 different values (no missing values).
ESTADO METEREOLÓGICO Text The whether status during the accident. 5 different options, 63 missing values
TIPO VEHÍCULO Text The type of the vehicle. 2 different values (no missing values).
TIPO PERSONA Text The type of the person (conductor, pasajero, peaton). 3 different values (no missing values).
RANGO EDAD Text The range of the person's age. 18 different values (no missing values). There no missing values, however, one of hte values is DECONOCIDE. This needs to be handled later.
SEXO Text The gender of the person. 2 different values, 41 missing values.
LESIVIDAD Number The harmfulness of the accident. It has a value between 1-15 based on how harmful the accident was. 265 missing values.

Resource naming strategy

Hash URIs for ontological terms, slash URIs for individuals.

  • Domain: http://bicycleaccidents.org/
  • Ontologycal terms: http://bicycleaccidents.org/ontology#
  • Individuals path: http://bicycleaccidents.org/resource/
  • Ontological terms example: http://bicycleaccidents.org/ontology#hasAge
  • Individuals example: http://bicycleaccidents.org/resource/2019S000036
